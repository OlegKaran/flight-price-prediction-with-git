# ✈️ Flight Price Prediction

## 📌 О проекте
Веб-приложение для прогнозирования стоимости авиабилетов с использованием алгоритмов машинного обучения. Модель обучена на реальных данных c Aviasales и учитывает такие факторы, как: даты, пересадки, маршруты и тд.

Проект включает в себя полный цикл ML-разработки: от сбора и предобработки данных до подбора гиперпараметров и вывода готовой модели в интерактивный веб-интерфейс.

## 🚀 Ключевые особенности
* **Продвинутое ML-моделирование:** Проведено сравнение нескольких алгоритмов (от ансамблей Random Forest до градиентного бустинга CatBoost, LightGBM) с использованием кросс-валидации (GridSearchCV).
* **Трекинг экспериментов:** Использован MLflow с локальной SQLite-базой для логирования лучших гиперпараметров, метрик качества: MAE, MAPE.
* **Удобный UI:** Интерактивный интерфейс реализован на Streamlit — пользователь может ввести параметры рейса и моментально получить прогноз цены.
* **Реализация REST API**: Интеграция модели в продакшен-среду с помощью FastAPI. Настроена строгая типизация и валидация входящих запросов при помощи Pydantic, корректная обработка ошибок (HTTPException) и автоматическая генерация интерактивной документации Swagger UI.

* ## 🧠 Работа с данными (Data Engineering)
Качественный прогноз невозможен без правильной подготовки признаков. В рамках проекта реализовано:
* **Интеграция реальных данных:** Использован датасет на основе реальных цен на билеты.
* **Предрасчет дистанций (`routes.json`):** Географические расстояния между аэропортами являются важным признаком для ML-модели. Чтобы не перегружать приложение сложными математическими вычислениями в реальном времени, все минимальные расстояния и время перелета для каждого из маршрутов были заранее найдены в датасете и сохранены в отдельный файл `routes.json`. Это радикально снизило время отклика интерфейса при генерации предсказания.
* **Воспроизводимость предобработки (Serialization):** Для предотвращения утечки данных (data leakage) и обеспечения идентичности обработки признаков при обучении и в продакшене:
  * Функции для обработки поступающих от пользователя данных были вынесены в отдельный файл `ml-service`, чтобы в дальнейшем импортировать эти функции в веб-интерфейс для обработки запросов пользователей
  * Числовые признаки масштабировались с помощью `StandardScaler`.
  * Категориальные признаки кодировались через `OneHotEncoder`.
  * Обученные объекты предобработки были **сериализованы и сохранены в файлы** (вместе с финальной моделью). Они были задействованы в веб-интерфейсе, и каждый запрос пользователя переводился в *pandas DataFrame*, после чего обрабатывался функцией *prepare_data*, содержащей в себе обработку определенных столбцов при помощи scaler и onehotencoder и только потом данные передавались в CatBoost.

* ## 📊 Моделирование и результаты
Процесс поиска оптимального алгоритма строился на последовательном усложнении моделей и логировании результатов:

* **Baseline-модель:** В качестве baseline была выбрана простая модель *(Линейная регрессия с L2-регуляризацией)*. Это позволило зафиксировать начальное значение ошибки: MAE = 4293 ₽, MAPE = 29,05% и оценить целесообразность применения более сложных алгоритмов.
* **Перебор архитектур:** Были протестированы различные ансамблевые методы, такие как Random Forest, LightGBM и CatBoost.
* **Трекинг экспериментов:** Подбор гиперпараметров с помощью GridSearchCV и сравнение метрик между алгоритмами фиксировались с помощью **MLflow**.
* **Итоговый выбор:** Наилучший результат показала модель **RandomForest** с финальной абсолютной ошибкой MAE = 1735 ₽, MAPE = 9,56 %, что превосходит показатели baseline-модели на 33 %. Модель CatBoost показала результат незначительно хуже MAE = 1743 ₽, MAPE = 9,75 %, так что из-за меньшего веса самой модели в веб-интерфейсе была использован именно она. 

## 🛠 Стек технологий
* **Язык программирования:** Python
* **Backend (REST API):** FastAPI, Uvicorn, Pydantic
* **Машинное обучение:** Scikit-Learn, CatBoost (и ансамблевые методы)
* **Инфраструктура и MLOps:** MLflow, SQLite *(локальная БД для трекинга)*
* **Frontend:** Streamlit
* **Анализ и обработка данных:** Pandas, Joblib *(сериализация пайплайнов)*
